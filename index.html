The Inner Loop

An execution framework for building machines that think.

We don’t build apps.
We build systems that produce outcomes.

Sometimes the system is code.
Sometimes it’s Markdown.
Sometimes it’s a set of rules, memory, and an LLM.

The medium doesn’t matter.
The output does.

Inner Loop is not a course.
Not a community for talking.
Not content to binge.

It is a repeatable way of working:

Define a constraint.
Design a machine.
Ship something real.
Reflect.
Repeat.

Over time, execution compounds.
Thinking becomes external.
Systems start to act on your behalf.
