The Method

A repeatable way of building machines that think.

The Inner Loop is not a framework you install. It is a method you apply.

Every machine — no matter how simple or complex — follows the same loop.

Define → Build → Ship → Reflect → Repeat

1. Define the Output

Every loop starts with a concrete output.

Not “learn something”.
Not “explore an idea”.
But a result that can be produced, tested, and finished.

2. Design the Machine

Before tools, define the structure.

A machine has:
• inputs
• rules
• memory
• output

Code is optional. Clarity is required.

3. Build the Smallest Version

Build the smallest version that works.

Ugly is acceptable. Incomplete is not.

4. Persist State

A machine that forgets does not compound.

State can live in:
• Markdown
• JSON
• LocalStorage
• Files

5. Ship

Run the machine.

If it produces the intended output, it ships. Perfection is deferred.

6. Reflect

Reflection turns execution into leverage.

Every loop records:
• what mattered
• what was cut
• what could scale

7. Repeat

The loop is run again.

Constraints change. Machines evolve. Judgment improves.

Over time, thinking moves out of the head and into systems.
